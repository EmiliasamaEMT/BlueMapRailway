# 使用文档

本文档面向已经部署好 BlueMap 的 Paper 服务器，说明如何安装和使用 BlueMapRailway。

## 前置条件

服务器需要满足：

- 已部署 Paper 26.1.2。
- 已安装并能正常访问 BlueMap。
- 服务端 Java 版本与 Paper 26.1.2 要求一致。
- 你有服务器文件访问权限，可以向 `plugins/` 目录放入插件 jar。

BlueMapRailway 不会替代 BlueMap。服务器必须先安装 BlueMap，BlueMapRailway 只负责给 BlueMap 增加铁路覆盖层。

## 下载插件

从 GitHub Release 下载最新的正式 jar：

```text
BlueMapRailway-版本号.jar
```

例如：

```text
BlueMapRailway-0.1.0.jar
```

不要下载 `sources.jar`、`javadoc.jar` 或源码压缩包。Paper 服务端需要的是插件 jar。

## 安装步骤

1. 停止服务器。
2. 确认 BlueMap 已经在 `plugins/` 目录中，并且服务器此前启动过 BlueMap。
3. 将 `BlueMapRailway-版本号.jar` 放入服务器的 `plugins/` 目录。
4. 启动服务器。
5. 首次启动后，插件会生成配置文件：

```text
plugins/BlueMapRailway/config.yml
```

6. 根据你的世界名称调整配置。
7. 执行 `/railmap reload` 或重启服务器，使配置生效。

## 配置世界

默认配置示例：

```yaml
worlds:
  world:
    enabled: true
    scan-radius: -1
```

`world` 必须与你服务器的世界名一致。常见世界名可能是：

```text
world
world_nether
world_the_end
```

如果你的主世界不是 `world`，需要改成实际名称。例如：

```yaml
worlds:
  survival:
    enabled: true
    scan-radius: -1
```

多个世界可以这样配置：

```yaml
worlds:
  world:
    enabled: true
    scan-radius: -1
  world_nether:
    enabled: false
    scan-radius: -1
  world_the_end:
    enabled: false
    scan-radius: -1
```

## 扫描范围说明

当前版本只扫描“已加载区块”，不会主动加载未加载区块。启用历史缓存后，玩家或服务器重新加载到某个区块时，插件会自动把该区块加入延迟扫描队列。

```yaml
scan-radius: -1
```

表示扫描该世界当前所有已加载区块。

```yaml
scan-radius: 2000
```

表示以世界出生点为中心，只扫描半径 2000 格范围内的已加载区块。

如果 BlueMap 上没有显示远处铁路，通常是因为那些铁路所在区块在插件安装后还没有被扫描过。让玩家靠近并加载这些区块后，插件会自动扫描并写入缓存；但插件仍然不会主动加载无人区域。

## 显示配置

铁路图层配置在 `markers` 下：

```yaml
markers:
  set-id: railways
  label: Railways
  default-hidden: false
  line-width: 5
  y-offset: 0.35
  depth-test-enabled: false
```

字段说明：

- `set-id`：BlueMap 内部 MarkerSet ID，通常不用改。
- `label`：BlueMap 菜单中显示的图层名称。
- `default-hidden`：是否默认隐藏铁路图层。
- `line-width`：铁路线宽。
- `y-offset`：线条相对铁轨方块的 Y 轴抬高量。
- `depth-test-enabled`：是否启用深度测试。默认 `false`，线条更容易看见。

颜色配置：

```yaml
colors:
  rail: "#9ca3af"
  powered-rail: "#facc15"
  powered-rail-inactive: "#a16207"
  detector-rail: "#ef4444"
  activator-rail: "#38bdf8"
```

## 扫描性能配置

```yaml
scanner:
  chunks-per-tick: 1
  update-debounce-ticks: 40
```

`chunks-per-tick` 表示每 tick 扫描多少区块。数值越大，扫描越快，但对主线程压力越高。

建议：

- 小服或测试服：`1` 到 `4`
- 大服：先保持 `1`

`update-debounce-ticks` 表示铁轨变化后延迟多少 tick 触发重扫。`40` tick 约等于 2 秒。这样可以把短时间内的多次放置、破坏、红石更新合并成一次扫描。

## 历史扫描缓存

插件默认启用历史扫描缓存：

```yaml
cache:
  enabled: true
  file: cache/rail-cache.yml
  scan-newly-loaded-chunks: true
  chunk-load-debounce-ticks: 100
```

工作方式：

1. 每次扫描时，插件先读取缓存中的历史铁轨。
2. 然后扫描当前已加载区块，或扫描由 `ChunkLoadEvent` 触发的指定区块。
3. 被扫描区块的扫描结果会覆盖对应缓存。
4. 扫描结束后，缓存和当前扫描结果一起输出到 BlueMap 和 SVG。

这意味着：

- 插件安装后，只要某个铁路区块被扫描过，之后没人加载也可以继续显示。
- `scan-newly-loaded-chunks: true` 时，玩家经过并加载到的区块会自动进入扫描队列，不需要执行玩家指令。
- `chunk-load-debounce-ticks` 用于合并短时间内的大量 chunk 加载事件，默认 `100` tick 约等于 5 秒。
- 如果某个已缓存区块再次被加载并扫描到铁轨已拆除，该区块会从缓存中清除。
- 插件不会主动加载旧区块，也不会自动读取未扫描过的 `.mca` 存档区域。
- 服务器上很久以前有人去过，但插件安装后从未扫描过的铁路，不会凭空出现在缓存中。

缓存文件默认位于：

```text
plugins/BlueMapRailway/cache/rail-cache.yml
```

如果想完全关闭缓存：

```yaml
cache:
  enabled: false
```

## 废弃矿井过滤

为了减少 BlueMap 上的干扰，插件默认启用两类过滤器。

第一类是全局零碎铁路过滤：

```yaml
filters:
  hide-short-lines: true
  short-line-max-points: 3
  short-line-max-length: 6.0
```

当一个连通铁路网络的点数不超过 `short-line-max-points`，或估算长度不超过 `short-line-max-length` 时，不会显示到 BlueMap。这个过滤不关心高度，也不关心铁轨类型，用于屏蔽自然生成、误放置、残留拆除等造成的短小线段。

正常铁路可以由一种或多种铁轨组成，也可以拐弯或上下坡。只要整个连通网络的连续长度超过阈值，就会保留；长铁路里短小的动力铁轨/探测铁轨片段不会单独被这个全局过滤器误删。如果你的服务器存在很短的站内支线、装卸点、红石机械短轨，可以关闭：

```yaml
filters:
  hide-short-lines: false
```

或者调小阈值：

```yaml
filters:
  short-line-max-points: 2
  short-line-max-length: 3.0
```

第二类是废弃矿井过滤。废弃矿井通常会在地下生成不完整的普通铁轨。为了减少这类干扰，插件默认启用组合过滤：

```yaml
filters:
  hide-fragmented-plain-rail-below-min-y: true
  min-y: 50
  fragmented-line-max-points: 8
  fragmented-line-max-length: 32.0
```

当一段铁路线同时满足以下条件时，不会显示到 BlueMap：

- 整段铁轨都低于 `min-y`，默认是 Y=50。
- 所在连通铁路网络只有普通 `RAIL`。
- 所在连通铁路网络没有 `POWERED_RAIL`、`DETECTOR_RAIL`、`ACTIVATOR_RAIL`。
- 线段较短碎，默认点数不超过 8，或估算长度不超过 32 格。

这个规则的目的是屏蔽大部分地下废弃矿井铁轨，同时尽量保留玩家铁路。Minecraft Wiki 对废弃矿井的描述是：普通矿井主要在地下生成，恶地矿井可以在更高海拔甚至地表生成。因此高度过滤只能作为启发式规则，不能完整识别所有废弃矿井。玩家地下铁路如果完全由普通铁轨组成、又比较短，也可能被过滤；这种情况下可以关闭：

```yaml
filters:
  hide-fragmented-plain-rail-below-min-y: false
```

或者降低限制：

```yaml
filters:
  min-y: 20
  fragmented-line-max-length: 16.0
```

## SVG 线路图导出

插件可以在每次扫描完成后导出一个地理型 SVG 文件，方便放到网页中展示或做二次加工。

默认配置：

```yaml
export:
  svg:
    enabled: true
    directory: export
    file: railways.svg
    title: BlueMap Railway
    padding: 16.0
    non-scaling-stroke: true
```

默认输出路径：

```text
plugins/BlueMapRailway/export/railways.svg
```

这个 SVG 使用 Minecraft 坐标做平面投影：

```text
Minecraft x -> SVG x
Minecraft z -> SVG y
```

Y 高度不会参与二维布局，但原始铁路在 BlueMap 图层中仍然按三维折线显示。SVG 中每条线会带有 `data-world`、`data-type` 和 `data-powered` 属性，网页端可以用这些属性做筛选、图例或交互。

如果想关闭导出：

```yaml
export:
  svg:
    enabled: false
```

如果想把文件名改成更适合网页使用的名字：

```yaml
export:
  svg:
    file: railway-map.svg
```

网页中可以直接引用：

```html
<object data="/railways.svg" type="image/svg+xml"></object>
```

## 线路命名与配色

插件会把互相连接的铁轨网络识别为 `RailComponent`，并为每个 component 生成稳定 ID。管理员可以通过 `routes.yml` 把一个或多个 component 归入同一条线路，并设置名称、颜色和线宽。

首次启动后会自动生成：

```text
plugins/BlueMapRailway/routes.yml
```

示例：

```yaml
version: 1

routes:
  main-line:
    name: "主线"
    color: "#22c55e"
    line-width: 6
    components:
      - "world:component:8e5f3a12abcd"
  mine-branch:
    name: "矿区支线"
    color: "#f97316"
    line-width: 5
    components:
      - "world:component:91ab20cd34ef"
```

配置步骤：

1. 执行 `/railmap debug`。
2. 从输出中找到需要归类的 component ID。
3. 编辑 `plugins/BlueMapRailway/routes.yml`。
4. 执行 `/railmap reload`。

被归入 route 的铁路线会使用 route 的名称、颜色和线宽。未归类的铁路线仍按铁轨类型使用默认颜色。

BlueMap 中会按线路拆分 MarkerSet：

```text
Railways - 主线
Railways - 矿区支线
Railways - 未分类
Railways - 站点
```

SVG 中也会按 route 分组：

```xml
<g id="route-main-line" data-route-id="main-line" data-route-name="主线">
```

当前 component ID 由世界名、边界、节点数量、长度和采样点生成。小范围调整线路通常不会影响无关 component，但大规模改造、拆分或合并铁路后，component ID 可能变化，需要重新绑定。

### 线路管理命令

列出线路：

```text
/railmap route list
```

查看线路详情：

```text
/railmap route info main-line
```

创建线路：

```text
/railmap route create main-line 主线
```

设置颜色：

```text
/railmap route color main-line #22c55e
```

设置线宽：

```text
/railmap route width main-line 6
```

将玩家附近最近的 component 绑定到线路：

```text
/railmap route assign-nearest main-line 16
```

`assign-nearest` 只能由游戏内玩家执行。半径参数可省略，默认 16 格。执行后插件会写入 `routes.yml` 并排队重扫。

## 站点区域

插件支持用 `stations.yml` 手动定义站点区域。首次运行后会自动生成：

```text
plugins/BlueMapRailway/stations.yml
```

示例：

```yaml
version: 1

stations:
  spawn:
    name: "出生点站"
    world: "world"
    area:
      type: box
      min: [120, 60, -30]
      max: [170, 75, 20]
```

当前第一版只使用 box 的 `min` 和 `max` 计算站点中心点。站点会显示为：

- BlueMap 中的 `Railways - 站点` POI marker。
- SVG 中的 `<circle>` 和站点名称 `<text>`。

修改 `stations.yml` 后执行：

```text
/railmap reload
```

## 管理命令

以下命令需要权限：

```text
bluemaprailway.admin
```

默认 OP 拥有该权限。

查看状态：

```text
/railmap status
```

查看调试信息和 component ID：

```text
/railmap debug
```

重载配置并排队重扫：

```text
/railmap reload
```

手动触发完整重扫：

```text
/railmap rescan
```

这些命令是管理员工具。插件正常运行时会自动扫描和更新，不需要普通玩家执行命令。

## 验证是否工作

启动服务器后，控制台应看到类似日志：

```text
BlueMap API 已就绪，铁路覆盖层服务已启动。
铁路扫描任务已创建，待扫描区块数: ...
铁路扫描完成: ... 个区块，... 个铁轨，... 条线。
```

进入游戏或控制台执行：

```text
/railmap status
```

然后打开 BlueMap 网页，检查图层菜单中是否出现：

```text
Railways
```

如果存在铁轨且所在区块已经加载，地图上应出现彩色铁路线。

## 调试演示线

如果想先验证 BlueMap 图层是否能显示，而服务器附近暂时没有铁轨，可以临时打开：

```yaml
debug:
  render-demo-line: true
```

然后执行：

```text
/railmap reload
```

BlueMap 会在启用世界的出生点附近生成一条演示线。确认显示正常后，应改回：

```yaml
debug:
  render-demo-line: false
```

## 常见问题

### BlueMap 菜单里没有 Railways

检查：

- BlueMap 是否正常启动。
- BlueMapRailway 是否已放入 `plugins/`。
- 控制台是否有插件加载错误。
- `config.yml` 中世界名是否正确。
- 是否执行过 `/railmap reload` 或重启服务器。

### 有铁轨但地图上没有线

当前版本只扫描已加载区块。请确认铁轨所在区块已经被服务器加载。

也可以临时开启：

```yaml
debug:
  render-demo-line: true
```

如果演示线能显示，说明 BlueMap 输出链路正常，问题大概率在扫描范围或区块加载状态。

### 扫描时担心卡服

降低：

```yaml
scanner:
  chunks-per-tick: 1
```

当前默认就是保守值 `1`。

### 修改配置后没有变化

执行：

```text
/railmap reload
```

或者重启服务器。

### 可以直接热重载插件吗

不建议使用 PlugMan 之类工具热重载服务端插件。推荐完整重启服务器，尤其是 BlueMap 相关插件。

## 卸载

1. 停止服务器。
2. 删除 `plugins/` 目录中的 BlueMapRailway jar。
3. 如不再需要配置，删除：

```text
plugins/BlueMapRailway/
```

4. 启动服务器。

如果 BlueMap 网页仍缓存旧图层，刷新浏览器页面或清理浏览器缓存。
