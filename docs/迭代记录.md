# 迭代记录

本文档记录每次开发迭代的更新内容、时间和下一步计划。

## 2026-05-12 0.1.5

### 更新内容

- 增加历史扫描缓存 `RailChunkCache`。
- 默认保存已扫描且包含铁轨的区块到 `plugins/BlueMapRailway/cache/rail-cache.yml`。
- 扫描开始时自动读取缓存，并与当前已加载区块扫描结果合并。
- 当前已加载区块会覆盖对应缓存；如果铁轨被拆除，该区块缓存会被清除。
- 增加 `ChunkLoadEvent` 监听，玩家或服务器加载新 chunk 后会自动加入区块扫描队列。
- 增加定点区块扫描模式，用历史缓存 + 新加载区块结果重新生成 BlueMap 覆盖层和 SVG。
- 增加 `cache.scan-newly-loaded-chunks` 和 `cache.chunk-load-debounce-ticks` 配置项。
- 增加 `docs/未来展望-Fabric支持.md`，记录 Fabric 1.21.11 支持的模块化改造方向。
- `/railmap status` 和 `/railmap debug` 增加缓存区块、缓存铁轨统计。
- 更新 README、使用文档和技术设计文档，说明缓存的自动执行和边界。

### 技术决策

- 缓存只保存有铁轨的区块，避免大量空区块膨胀缓存文件。
- 第一版缓存使用 YAML 文件，便于服主查看和排查。
- 缓存不主动加载区块，不读取 `.mca`，优先避免卡服风险。
- `ChunkLoadEvent` 只触发对应 chunk 的延迟扫描，不触发完整重扫，避免玩家移动时反复扫描所有已加载区块。
- 完整重扫会取消尚未执行的 chunk-load 局部扫描队列，避免重复扫描。

### 验证

- 已通过 `gradlew build`。

### 下一步计划

- 观察大型服务器上 YAML 缓存文件体积，再考虑拆分为按世界或按 region 的缓存文件。
- 增加缓存清理/重建命令。

## 2026-05-12 0.1.4

### 更新内容

- BlueMap 输出按线路拆分 MarkerSet：
  - `railways.route.<route-id>`
  - `railways.unclassified`
  - `railways.stations`
- SVG 导出按 route 增加 `<g>` 分组，方便网页端按线路控制显示。
- 增加 `/railmap route` 管理命令：
  - `list`
  - `info`
  - `create`
  - `color`
  - `width`
  - `assign-nearest`
- `assign-nearest` 支持玩家站在铁路旁把最近 component 绑定到指定线路。
- 增加 `stations.yml`，支持手动配置站点 box 区域。
- BlueMap 中将站点显示为 POI marker。
- SVG 中将站点显示为圆点和站点名称。
- 站点 POI 详情文本增加 HTML 转义，避免配置中的特殊字符影响 BlueMap 弹窗。
- 更新 README、使用文档和技术设计文档。

### 技术决策

- 线路 MarkerSet 使用扁平 ID 命名约定，而不是模拟嵌套结构。
- 站点第一版只做显示标注，不改变铁路图构建和线路切分。
- route 命令直接写入 `routes.yml`，写入后排队重扫以复用现有渲染流程。

### 验证

- 已通过 `gradlew build`。
- 已通过 `gradlew build -PpluginVersion=0.1.4`。

### 下一步计划

- 在真实服务器验证 route MarkerSet 数量和 BlueMap 菜单体验。
- 增加 `route unassign` 和更友好的 component 选择流程。
- 让站点区域参与线路切分，避免站场内部复杂股道干扰主线识别。

## 2026-05-12 0.1.3

### 更新内容

- 改进铁轨连接判断，构建邻接图时增加双向兼容检查，降低复杂线路附近的误连接概率。
- 引入 `RailComponent`，扫描后显式生成连通铁路分量。
- 为每个 component 生成稳定 ID，供调试和线路归类使用。
- 增加 `RailGraphResult`，记录可见线段、component 列表和被过滤隐藏的线段数。
- 增加 `/railmap debug`，输出扫描统计、过滤统计、SVG 输出路径和前 8 个 component ID。
- 增加 `routes.yml`，支持按 component ID 给线路命名、设置颜色和线宽。
- BlueMap `LineMarker` 和 SVG 导出都会应用 route 样式。
- SVG 线条新增 `data-component-id`，已归类线路额外新增 `data-route-id` 和 `data-route-name`。
- 更新 README、使用文档和技术设计文档，补充 component、debug 和 `routes.yml` 使用说明。

### 技术决策

- 扫描层保持物理铁路识别，线路命名和配色由 `RailRouteRegistry` 在扫描结果后处理阶段应用。
- component ID 基于世界名、节点数量、估算长度、边界盒和采样节点生成，兼顾稳定性与实现复杂度。
- `routes.yml` 第一版仅支持手动绑定 component ID，不自动推断主线或站点。

### 验证

- 已通过 `gradlew build`。
- 已通过 `gradlew build -PpluginVersion=0.1.3`。

### 下一步计划

- 增加 route 管理命令，减少手写 component ID 的成本。
- 考虑按线路拆分 BlueMap MarkerSet 或 SVG 分组。
- 增加手动站点区域和站点 marker。

## 2026-05-12 0.1.2

### 更新内容

- 增加地理型 SVG 导出，扫描完成后默认生成 `plugins/BlueMapRailway/export/railways.svg`。
- SVG 使用 Minecraft `x/z` 坐标投影为二维线路图，每条铁路线输出为 `<polyline>`。
- SVG 线条保留 `data-world`、`data-type`、`data-powered` 属性，方便网页端做筛选、图例和交互。
- 增加 `export.svg` 配置项，可控制是否导出、输出目录、文件名、标题、边距和线宽缩放行为。
- 修正 SVG 导出路径创建逻辑，支持 `export.svg.file` 中包含子目录。
- 将 Java 包名、Gradle group 和 `plugin.yml` 主类从临时命名改为 `io.github.emiliasamaemt.bluemaprailway`。
- 更新 `plugin.yml` 作者和仓库地址。
- 更新 README、使用文档和技术设计文档，补充 SVG 导出说明。

### 技术决策

- SVG 导出复用同一次 `RailScanResult`，不重复扫描世界。
- 当前 SVG 是真实坐标投影，不做地铁线路图式自动布局。
- 当前多世界结果写入同一个 SVG，并通过世界分组区分；后续可按世界拆分独立 SVG。

### 更新内容

- 增加未来展望文档 `docs/未来展望-线路管理.md`，规划线路分类、命名、配色、持久化和管理员交互。
- 补充未来展望：站点汇聚区域、站内轨道、双轨走廊、真实轨道/中心线显示模式。
- 增加面向服主的中文使用文档 `docs/使用文档.md`。
- 创建独立 Paper 插件工程 `BlueMapRailway`。
- 增加 Gradle Wrapper，支持无需本机安装 Gradle 直接构建。
- 增加 GitHub Actions：
  - `build.yml`：主分支和 PR 构建检查。
  - `release.yml`：推送 `v*` tag 时自动发布 GitHub Release。
- 增加 `.gitattributes`，声明文本文件使用 UTF-8。
- 接入 BlueMap API 生命周期：
  - `BlueMapAPI.onEnable`
  - `BlueMapAPI.onDisable`
- 增加管理员命令：
  - `/railmap status`
  - `/railmap reload`
  - `/railmap rescan`
- 增加中文默认配置 `config.yml`。
- 增加铁路数据模型：
  - `RailType`
  - `RailPosition`
  - `RailDirection`
  - `RailNode`
  - `RailConnection`
  - `RailLine`
  - `RailScanResult`
- 增加铁轨读取器 `RailBlockReader`，支持 4 种原版铁轨和 `Rail.Shape`。
- 增加图构建器 `RailGraphBuilder`，将连续铁轨合并成 `RailLine`。
- 增加扫描器 `RailScanner`，按配置世界分批扫描已加载区块。
- 增加地理型 SVG 导出器 `SvgRailExporter`，扫描完成后生成 `plugins/BlueMapRailway/export/railways.svg`。
- 增加全局零碎铁路过滤，默认隐藏点数不超过 3 或长度不超过 6 格的短线段。
- 修正全局零碎铁路过滤，使其按连通铁路网络整体判断，避免误删长铁路里的短动力铁轨/探测铁轨片段。
- 增加 `RailLineFilter`，默认屏蔽 Y=50 以下短碎、纯普通铁轨的疑似废弃矿井线段。
- 增加 BlueMap 渲染器 `BlueMapRailRenderer`，将铁路线写成 `LineMarker`。
- 增加事件监听器 `RailwayBlockListener`，在铁轨变化后延迟触发重扫。
- 将 README 改为中文，并新增技术设计文档。

### 技术决策

- 当前版本只使用 BlueMap API marker 覆盖层，不修改 BlueMap 内部渲染流程。
- SVG 导出复用扫描结果，不重复扫描世界；当前只做真实坐标投影，不做示意图自动布局。
- 当前版本不主动加载未加载区块，优先避免服务器启动时卡顿。
- 当前版本扫描运行在 Bukkit 主线程，通过 `chunks-per-tick` 控制压力。
- 废弃矿井过滤采用启发式组合判断，不依赖结构数据。
- 使用 Paper 当前主线依赖版本 `26.1.2.build.+`，Java toolchain 设置为 25。
- 发布产物只包含本插件代码和资源，不打包 Paper API 或 BlueMap API。
