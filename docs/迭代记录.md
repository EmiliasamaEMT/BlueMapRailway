# 迭代记录

本文档记录每次开发迭代的更新内容、时间和下一步计划。

## 2026-05-12 0.1.0-SNAPSHOT

### 更新内容

- 增加未来展望文档 `docs/未来展望-线路管理.md`，规划线路分类、命名、配色、持久化和管理员交互。
- 补充未来展望：站点汇聚区域、站内轨道、双轨走廊、真实轨道/中心线显示模式。
- 增加面向服主的中文使用文档 `docs/使用文档.md`。
- 创建独立 Paper 插件工程 `BlueMapRailway`。
- 增加 Gradle Wrapper，支持无需本机安装 Gradle 直接构建。
- 增加 GitHub Actions：
  - `build.yml`：主分支和 PR 构建检查。
  - `release.yml`：推送 `v*` tag 时自动发布 GitHub Release。
- 增加 `.gitattributes`，声明文本文件使用 UTF-8。
- 接入 BlueMap API 生命周期：
  - `BlueMapAPI.onEnable`
  - `BlueMapAPI.onDisable`
- 增加管理员命令：
  - `/railmap status`
  - `/railmap reload`
  - `/railmap rescan`
- 增加中文默认配置 `config.yml`。
- 增加铁路数据模型：
  - `RailType`
  - `RailPosition`
  - `RailDirection`
  - `RailNode`
  - `RailConnection`
  - `RailLine`
  - `RailScanResult`
- 增加铁轨读取器 `RailBlockReader`，支持 4 种原版铁轨和 `Rail.Shape`。
- 增加图构建器 `RailGraphBuilder`，将连续铁轨合并成 `RailLine`。
- 增加扫描器 `RailScanner`，按配置世界分批扫描已加载区块。
- 增加全局零碎铁路过滤，默认隐藏点数不超过 3 或长度不超过 6 格的短线段。
- 修正全局零碎铁路过滤，使其按连通铁路网络整体判断，避免误删长铁路里的短动力铁轨/探测铁轨片段。
- 增加 `RailLineFilter`，默认屏蔽 Y=50 以下短碎、纯普通铁轨的疑似废弃矿井线段。
- 增加 BlueMap 渲染器 `BlueMapRailRenderer`，将铁路线写成 `LineMarker`。
- 增加事件监听器 `RailwayBlockListener`，在铁轨变化后延迟触发重扫。
- 将 README 改为中文，并新增技术设计文档。

### 技术决策

- 当前版本只使用 BlueMap API marker 覆盖层，不修改 BlueMap 内部渲染流程。
- 当前版本不主动加载未加载区块，优先避免服务器启动时卡顿。
- 当前版本扫描运行在 Bukkit 主线程，通过 `chunks-per-tick` 控制压力。
- 废弃矿井过滤采用启发式组合判断，不依赖结构数据。
- 使用 Paper 当前主线依赖版本 `26.1.2.build.+`，Java toolchain 设置为 25。
- 发布产物只包含本插件代码和资源，不打包 Paper API 或 BlueMap API。
