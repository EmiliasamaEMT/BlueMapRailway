# 技术设计

本文档记录 BlueMapRailway 的实现方案。随着版本迭代，新增模块、关键决策和风险都应继续补充到这里。

## 目标

BlueMapRailway 的目标是在 Paper 26.1.2 服务端自动维护一个 BlueMap 铁路覆盖层，用于显示原版 Minecraft 的 4 种铁轨：

- `RAIL`
- `POWERED_RAIL`
- `DETECTOR_RAIL`
- `ACTIVATOR_RAIL`

插件不修改 BlueMap 渲染器本体，而是通过 BlueMap API 写入 `MarkerSet` 和 `LineMarker`。

## 当前架构

```text
BlueMapRailwayPlugin
  -> RailwayService
      -> RailScanner
          -> RailBlockReader
          -> RailGraphBuilder
      -> BlueMapRailRenderer
  -> RailwayBlockListener
```

### BlueMapRailwayPlugin

主插件类，负责：

- 保存默认配置。
- 注册 `RailwayBlockListener`。
- 注册 BlueMap API 生命周期回调。
- 处理 `/railmap` 管理命令。

### RailwayService

服务调度层，负责：

- 保存当前 `BlueMapAPI` 实例。
- 管理扫描任务和延迟重扫任务。
- 调用 `RailScanner` 分批扫描。
- 调用 `BlueMapRailRenderer` 写入 BlueMap。
- 提供状态文本给管理员命令。

### RailBlockReader

铁轨识别层，输入 Bukkit 方块数据：

```text
World + x/y/z + Material + BlockData
```

输出内部 `RailNode`。它会读取：

- 铁轨类型：`RailType`
- 坐标：`RailPosition`
- 形态：`Rail.Shape`
- 是否通电：`Powerable#isPowered`
- 连接方向：`RailDirection`

### RailGraphBuilder

图构建层，输入 `Map<RailPosition, RailNode>`，输出 `List<RailLine>`。

当前策略：

- 根据 `Rail.Shape` 计算每个节点的连接方向。
- 生成无向邻接表。
- 从端点或分叉点开始行走。
- 将连续、同类型、同通电状态的铁轨合并成一条 `RailLine`。
- 对纯环路做第二轮兜底遍历。

当前实现仍是 MVP 级别。后续需要重点改进斜坡连接、跨 chunk 边界缓存和复杂分叉线路的命名能力。

### RailScanner

扫描层，当前只扫描配置世界中的已加载区块。

`scan-radius` 规则：

- `-1`：扫描该世界当前所有已加载区块。
- `>= 0`：以世界出生点为中心，扫描半径内已加载区块。

当前版本刻意不主动加载未加载区块，避免服务器启动时造成额外卡顿。

### BlueMapRailRenderer

BlueMap 输出层，负责：

- 按 `worldName` 分组 `RailLine`。
- 创建 `MarkerSet`。
- 将每条 `RailLine` 转成 `LineMarker`。
- 写入该 BlueMap world 下的所有 `BlueMapMap`。

渲染配置包括：

- MarkerSet ID
- MarkerSet 标签
- 默认隐藏状态
- 线宽
- Y 轴抬高量
- 深度测试开关
- 各类铁轨颜色

## 线程模型

当前扫描任务运行在 Bukkit 主线程，并通过 `chunks-per-tick` 控制每 tick 处理区块数。

这样做的原因：

- Bukkit 世界与方块 API 通常不应在异步线程访问。
- MVP 阶段优先保证正确性和服务端安全。

后续如果要支持全图扫描，应优先考虑 `ChunkSnapshot` 或直接读取 `.mca`，但那会引入额外复杂度。

## 自动更新模型

当前监听以下事件：

- `BlockPlaceEvent`
- `BlockBreakEvent`
- `BlockPhysicsEvent`
- `BlockRedstoneEvent`

当事件涉及铁轨时，不立即扫描，而是根据 `scanner.update-debounce-ticks` 延迟触发完整重扫。短时间内的多次变化会被合并。

后续计划改成 chunk/region 级 dirty 标记，只重算受影响区域。

## 已知限制

- 只扫描已加载区块，不会主动发现离线区域中的铁轨。
- 事件更新目前触发完整重扫，不是局部增量。
- 斜坡连接只做基础支持，复杂高度关系需要在真实服务器里验证。
- 没有持久化缓存，重启后需要重新扫描。
- 没有线路命名、站点、颜色分组等高级铁路网络功能。

## 下一阶段方向

- 使用 `ChunkSnapshot` 降低主线程扫描压力。
- 引入 chunk 级缓存和 dirty 更新。
- 将扫描结果持久化到插件数据目录。
- 支持管理员配置不同世界的扫描范围。
- 增加站点 marker 和线路命名。
